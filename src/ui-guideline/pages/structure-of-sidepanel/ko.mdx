# Side Panel의 구조

## 안내문, 경고문과 같은 메세지 넣기

Panel에는 총 2가지 방식으로 메세지를 넣을 수 있습니다.

<MessageTypesExample width={width} height={300} />

|방식        |사용자가 인지하기 쉬운가?                             |공간을 많이 차지할 수 있는가?     |사용예       |
|:----------|:-----------------------------------------------|:--------------------------|:----------|
|Tooltip    |사용자가 마우스를 올려야만 보이기 때문에 전달력은 떨어집니다.  |공간을 적게 차지하는 편입니다.    |사용자가 바로 읽을 필요없는 보조적 안내, 또는 Panel에 내용이 지나치게 많아서 메세지를 넣기 어려운 경우 |
|본문에 표현   |눈에 바로 보이기 때문에 전달력이 좋습니다.                |공간을 많이 차지하는 편입니다.    |Panel에 공간이 충분히 있다고 생각되는 모든 경우, 사용자에게 직접적으로 전달되어야 하는 중요한 메세지   |

일관성을 고려해서 Tooltip을 주로 사용한다면 가능한 Tooltip 방식만, 본문에 포함시키는 방식이 주로 사용된다면 본문에 포함시키는 방식만 사용하는 것이 좋습니다.

일관성의 유지는 사용자가 UI에 익숙해진 이후, 어떤 기능을 유추할 수 있게 해줍니다. 

특별한 경우가 아니라면 기능 보다는 일관성을 더 우선적으로 고려하는 것이 좋습니다.

## 메세지의 레벨 (Level)

Message는 기본적으로 4가지 레벨을 가지고 있습니다.

<MessageLevelExample width={width} height={480} />

특별한 경우 별도의 다른 Style을 지정할 수도 있지만, 일관성을 고려해서 가능한 4가지 레벨 중 한 가지로 표현하는 것이 좋습니다.

## 접기

필요에 따라 Side Panel을 접을 수 있습니다.

접은 상태는 브라우저에 저장되기 때문에 새로고침을 해도 접은 상태가 계속 유지됩니다. 

<FoldingExample width={width} height={600} />

1. 사용자가 UI에 익숙하지 않은 상황 (초기 상태)에서는 좀 더 쉽게 이해할 수 있는 형태로 모양을 제공
2. 사용자가 UI에 익숙해진 이후, 접어서 Side Panel을 작게 사용

위와 같이 사용자의 UI 학습 시나리오를 고려하고자 하는 경우 사용할 수 있습니다.

굳이 필요하지 않다고 판단되는 경우 가능한 사용하지 않은게 좋습니다. (사용자가 UI에 대해 유추해야 하는 것들을 늘려서 불필요한 학습을 발생시킵니다)
